# Todos

This directory tracks deferred work, technical debt, and future improvements.

## Structure

Each todo is a markdown file with:
- YAML frontmatter (id, title, priority, category, status, dates, labels)
- Description and context
- Affected files and acceptance criteria

## Priority Levels

| Priority | Description |
|----------|-------------|
| `high` | Address soon |
| `medium` | Next opportunity |
| `low` | When time permits |

## Categories

`tech-debt` | `refactor` | `feature` | `bug` | `performance` | `docs` | `test`

## Usage

See [create-todo workflow](../workflows/create-todo.prompt.md) for creating new todos.
