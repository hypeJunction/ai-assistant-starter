# History

This directory contains records of significant work completed on the project.

## Purpose

Create history entries for:
- Refactors affecting 5+ files
- Architectural changes or pattern migrations
- Bug fixes with non-obvious solutions
- Feature implementations with design decisions

## Why Document History

- Preserve context for future reference
- Explain "why" behind changes
- Help new team members understand evolution
- Prevent repeating mistakes

## File Naming

Use descriptive names with type prefix:
- `refactor-api-client.md`
- `feature-user-auth.md`
- `bugfix-race-condition.md`
- `migration-v2-types.md`

## Template

See [_template.md](./_template.md) for the standard format.

## What to Include

1. **Summary** - What was accomplished
2. **Context** - Why was this needed
3. **Approach** - Decisions made and reasoning
4. **Patterns** - Before/after code examples
5. **Files** - What was modified/created/deleted
6. **Validation** - How correctness was verified
7. **Lessons** - Insights for future work
