# Changelog Format

When `CHANGELOG.md` exists, update it with the following format:

```markdown
## [X.Y.Z] - YYYY-MM-DD

### Breaking Changes
- Description ([commit])

### Added
- Description ([commit])

### Fixed
- Description ([commit])

### Changed
- Description ([commit])
```

If no `CHANGELOG.md` exists, release notes will be generated from commits instead.
